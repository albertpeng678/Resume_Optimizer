# Project Plan: Resume Forge - Extraction Core (隱性知識挖掘機)

## 1. 產品脈絡與痛點定義 (Context & Pain Points)

**核心使命：解決「執行鴻溝」，將履歷診斷轉化為代工開採。**
目前市場上的 AI 履歷工具僅停留在「診斷階段」（指出數據不足），本產品則定位為「隱性知識挖掘機」。我們透過主動引導與非同步 Agent 協作，將使用者模糊的記憶碎片，提煉為具備商業價值、且可溯源的結構化 JSON 資料。

* **解決痛點一：知識萃取的執行障礙** ➔ 使用者不知如何量化，系統必須提供公式填空而非開放式申論。
* **解決痛點二：靈魂拷問的流失率** ➔ 透過「選擇題優先」與「三振出局」降低使用者認知負荷。
* **解決痛點三：數據失真的恐懼** ➔ 提供「數據溯源（Traceability）」，確保每個數字都有推導根據。

## 3. 核心 Agent 規格與理想範例 (Specs & Ideal Trace)

本系統由兩個非同步協作的 Agent 組成：

### M1: 訪談助手 (The Inquisitor) - [主對話流]
* **設計核心：** 文本驅動、主動出擊、引用原文。
* **行為規格：**
    1.  **主動開局：** 讀取解析後的履歷 Markdown，針對目標 Persona 缺乏的 Gap 段落直接引用原文提問。
    2.  **互動策略：** 優先提供選項（Multiple Choice）或假設性情境。
    3.  **退場機制 (3-Turn Limit)：** 針對單一 Gap，追問上限為 2 次，若無果則標記 `skipped` 並進入下一話題。

> **💡 M1 理想執行範例：**
> * **System Input:** 解析後的履歷段落：「負責內部搜尋系統優化，變得很準且變快。」
> * **M1 Response (主動開場):** 「你好！我仔細看了你的經歷。針對『內部搜尋系統優化』，你當時是導入哪種技術來解決痛點的？
>     [ ] 導入 RAG / 向量檢索技術
>     [ ] 優化了關鍵字權重與標籤系統
>     [ ] 重構了底層查詢邏輯」

### M2: 量化軍師 (The Formulator Sidekick) - [併行輔助流]
* **設計核心：** 背景監聽、側邊欄公式填空、主動兜底。
* **行為規格：**
    1.  **公式推導面板 (Guided Formula Builder)：** 監聽主對話。嗅到量化訊號時，在側邊欄非同步生成「變數填空題」（如：輸入優化前後耗時）。
    2.  **主動兜底 (Safety Net)：** 若 M1 準備結束該主題且尚未觸發量化，M2 必須強制彈出側邊欄，提供該職位常見的 KPI 選項（如：節省工時、降低錯誤率）進行盲猜引導。
    3.  **數據溯源封裝 (Trace Metadata)：** 將「填寫變數」與「推導邏輯」打包成 `trace_log`。

> **💡 M2 理想執行範例與 JSON 產出：**
> *(User 在 M1 點選「導入 RAG」，M2 在側邊欄同步顯示：)*
> * **M2 (Sidekick):** 「💡 我們來推導數據。請預估：過去單次查詢 [ 3 ] 秒 ➔ 現在 [ 0.5 ] 秒。」
>
> **⚙️ 最終開採輸出的 JSON Payload (本階段交付物):**
> ```json
> {
>   "action_context": {
>     "original_text": "負責內部搜尋系統優化",
>     "solution": "導入 RAG / 向量檢索技術"
>   },
>   "quantified_metrics": [
>     {
>       "result": "效能提升 6 倍",
>       "trace_log": {
>         "formula": "舊耗時 / 新耗時",
>         "input": {"old": "3s", "new": "0.5s"}
>       }
>     }
>   ]
> }
> ```

## 4. 系統工作流 (Parallel Workflow)

```mermaid
graph TD
    Input[Parsed Resume & Persona Gaps] --> M1[M1 Inquisitor: 引用原文主動提問]
    M1 -- 提供選項引導 --> User[User 點擊回答]
    User -- 回答內容 --> M2[M2 Sidekick: 背景非同步監聽]
    M2 -- 偵測到量化訊號 --> SideUI[側邊欄: 彈出公式填空表單]
    M2 -- 若訪談即將結束且無數據 --> SideUI[側邊欄: 啟動兜底 KPI 選項]
    SideUI -- 填寫變數 --> State[狀態機整合]
    State -- 執行溯源封裝 --> Output[產出結構化 JSON Payload + Trace Log]
